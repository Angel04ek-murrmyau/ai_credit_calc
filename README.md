# ai_credit_calc
Проектная работа кредитного калькулятора на основе ИИ.

# Цель: Создание веб-приложения на Flask, которое принимает данные клиента и выдаёт кредитный вердикт.
## Логика проекта
    1. Ввод данных клиента через веб-интерфейс (пол, возраст, семейное положение, и т.д.)
    2. Обработка данных через формулу кредитного риска.
    3. Обработка данных нейросетевой моделью (локальная нейросеть через roocode, или api).
    4. Вердикт: кредит одобрен/отклонён, а также более подробное описание от нейросети.
    5. Создание Docker образа с проектом.
    6. Развертвыание приложения в Kubernetes с помощью minikube или kubeadm.
    7. Интеграция CI/CD с помощью Jenkins (стадии Build, Test, Deploy).
    8. Мониторинг и логирование с помощью Prometheus + Grafana + Grafana Loki.

## Технологии
    1. Python 3.14
    2. Flask
    3. Локальная LLM модель/модель через API
    4. Jenkins
    5. Prometheus + Grafana + Grafana Loki
    6. Minikube/Kubeadm
    7. SQLite/PostreSQL/Json для хранения данных о клиентах
    8. Docker 

## Модель данных
### Информация о клиенте
    1. id (int) - идентификационный номер клиента
    2. age (int) - возраст клиента
    3. required_sum (int) - требуемая клиентом сумма, руб.
    4. gender (str) - пол клиента
    5. status (str) - семейное положение
    6. job (str) - место работы клиента
    7. job_exp (int) - опыт работы клиента
    8. existing_loans (str) - текущие долги клиента
    9. income_score (int) - уровень дохода клиента

### Заявка
    1. id (int) - идентификационный номер заявки
    2. client_id (int) - идентификационный номер клиента
    3. risk_score (int) - уровень кредитного риска
    4. status (str) - статус заявки
    5. ai_comment (str) - пояснение от нейросети

### Расчёт
   1. id (int) - идентификационный номер расчёта
   2. request_id (int) - идентификационный номер заявки
   3. score (int) - уровень кредитного риска клиента
   4. required_score (int) - требуемый уровень кредитного риска клиента

## Расчёт кредитного риска
    ``` 
    risk_score = 
    0.2 * age_score +
    0.2 * income_score +
    0.15 * job_score +
    0.15 * job_exp_score +
    0.2 * credit_load_score +
    0.1 * marital_score
    ```
### Возраст (age_score)
    ```
    < 18 → 0 (автоматический отказ)
    18–21 → 0.2
    22–30 → 0.6
    31–50 → 1.0
    51–65 → 0.7
    > 65 → 0.3
    ```
### Уровень дохода (income_score)
    ```
    < 1000 → 0.2
    1000–3000 → 0.5
    3000–6000 → 0.8
    > 6000 → 1.0
    ```
    
### Место работы (job)
    ```
    безработный → 0
    временная работа → 0.3
    частичная занятость → 0.5
    официальная работа → 0.8
    госслужащий / IT / стабильная компания → 1.0
    ```

### Опыт работы (job_exp_score)
    ```
    < 6 месяцев → 0.2
    6–12 месяцев → 0.4
    1–3 года → 0.7
    > 3 лет → 1.0
    ```
### Кредитная нагрузка (credit_load_score)
    ```
    credit_load_score = existing_loans / income_score
    ```
    ```
    > 0.5 → 0.1
    0.3–0.5 → 0.4
    0.1–0.3 → 0.7
    < 0.1 → 1.0
    ```

### Семейное положение
    ```
    не женат / не замужем → 0.5
    женат / замужем → 0.8
    разведён → 0.6
    есть дети → +0.1 
    ```

### Итоговое решение
    ```
    risk_score < 0.4 - отказ
    risk_score >= 0.5 - одобрение
    ```

## Роль нейросетевой модели
Нейросетевая модель не принимает решение напрямую, а используется для генерации пояснений.
Входные данные:
```
{
  "age": 28,
  "income_score": 3200,
  "job": "IT",
  "risk_score": 0.72
}
```
Итог:
Клиент имеет низкий уровень кредитного риска, кредит одобрен.

## Ручной этап Ansible плейбука (Важно!)
На этапе применения Ansible плейбука main.py с ролями kubernetes и service будет один ручной этап. Когда плейбук попросит обратиться в документацию, подключитесь по ssh к master ноде (```ssh ubuntu@<IP master ноды, который вы скопировали в inventory.ini файле>```), и выполните ```sudo kubeadm init --pod-network-cidr=10.244.0.0/16```. После этого, на master ноде необходимо выполнить следующие команды: ```mkdir -p $HOME/.kube``` ```sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config``` ```sudo chown $(id -u):$(id -g) $HOME/.kube/config```. Они описаны в выводе команды kubeadm init. Также в результате вывода этой команды Вы получите команду вида ```kubeadm join…``` и токен. Скопируйте команду полностью, подключитесь по ssh к двум worker нодам (ssh ubuntu@<IP worker ноды, который Вы скопировали в inventory.ini файле>). Выполните её, прописав в начале «sudo» на worker ноде. Для этой команды необходимы права суперпользователя. После этого нажмите Enter и плейбук продолжит выполняться в автоматическом режиме.

## Ручной этап после завершения Jenkins job (Важно!)
После завершения Jenkins job, подключившись по ssh к master серверу, в консоли нужно выполнить ```kubectl apply -f /home/ubuntu/k8s/k8s.yml``` и ```kubectl rollout restart deployment ai-credit-calc```, эти команды применяет k8s манифесты, и принудительно их развернут, перезапустив поды.
